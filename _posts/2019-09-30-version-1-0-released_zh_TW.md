兩年前，我們公佈了引領我們走到這一刻的路線圖。雖然花費的時間比我們預想的要長，但現在 Node-RED 1.0 終於正式發佈了。

非常感謝社群中為此版本做出貢獻的每一個人。無論是透過程式碼、文件、想法還是反饋，這都有助於保持專案朝正確的方向發展。

由於此版本帶來了主版本號的提升，它包含了一些在升級之前您需要注意的變更。

### 非同步訊息傳遞

此版本將節點之間的訊息傳遞更改為始終是非同步的，而不是根據各個節點的實作有時是非同步、有時是同步的。

在某些情況下，這將在流程產生分支時改變訊息的相對順序。

有關此變更的更多詳細資訊，請閱讀[這篇部落格文章](https://nodered.org/blog/2019/08/16/going-async_zh_TW)。

### 節點發送 API

節點現在有一個新的 API 可用於處理執行環境中的訊息。此 API 允許它們告訴執行環境何時處理完了一條訊息。

這將幫助執行環境追蹤訊息在系統中的流動情況 — 從而實現未來的功能，例如節點自動逾時。

為了配合這項變更，我們引入了一個新的 Complete 節點。此節點可以針對另一個節點（類似 Catch 節點），當目標節點處理完訊息時，它將被觸發。

這僅適用於更新至新 API 的節點。

[這篇部落格文章](https://nodered.org/blog/2019/09/20/node-done_zh_TW)詳細介紹了新風格的 API。文件也已同步更新。

### 訊息複製 (Cloning)

非同步訊息傳遞變更的一個連鎖反應是 Function 節點複製訊息方式的變更。以前它會盡可能避免複製訊息，但自從引入非同步訊息變更後，這導致了難以察覺的問題。

因此，現在 Function 節點也會複製您傳遞給 `node.send()` 的所有訊息。

如果您的流程依賴於訊息不被複製，您需要更新流程以要求執行環境不要進行複製。

有關此內容的更多詳細資訊，請閱讀[這篇部落格文章](https://nodered.org/blog/2019/09/13/cloning-messages_zh_TW)。


### 編輯器功能

編輯器有許多值得關注的新功能：

#### 重組的調色盤

我們重組了調色盤，以提供我們認為更好的節點順序。

類別現在如下：
 - *common* (常用) — 基礎構件節點：Inject、Debug、Catch、Status、Links、Comment。
 - *function* (功能) — 主要的功能節點：Function、Switch、Change、Template、Exec、Delay、Trigger、Range。
 - *network* (網路) — 執行網路操作的節點。這意味著 HTTP In 和 HTTP Response 節點現在坐在一起，不再被拆分到不同類別 — MQTT、TCP 和 UDP 節點也是如此。
 - *sequence* (序列) — 用於處理訊息序列的節點：Split、Join、Sort 和 Batch。
 - *parser* (解析器) — 用於處理特定資料格式的節點：CSV、HTML、JSON、XML 和 YAML。
 - *storage* (儲存) — 檔案節點。

#### 已移除的節點

 **我們已從預設調色盤中移除了某些節點。**

**Twitter**、**Email**、**Feedparser**、**Sentiment** 和 **Pi GPIO** 節點不再作為 `node-red` 模組的依賴項安裝。它們在很早之前就已存在於各自的 npm 模組中，但如果您正在使用它們，則需要明確地將它們重新添加回來。

如果您使用我們的指令碼在 Raspberry Pi 上安裝，則 GPIO 節點已經為您重新安裝。


#### 子流程實例屬性 UI

在 0.20 中，我們添加了讓子流程範本定義一組環境變數的能力，並允許子流程的各個實例覆蓋其值。

在此版本中，當您定義子流程的環境變數時，還可以定義一個自定義 UI，以指定環境變數在實例節點中的呈現方式。

例如，您可能希望在子流程中設定一個布林標誌，以切換每個實例的行為。與其在實例節點中使用自由格式的文字欄位來設定標誌，您可以將其配置為顯示一個核取方塊。

您現在還可以自定義子流程節點的顏色。


#### 動作列表 (Action List)

您在編輯器中可以執行的許多操作都被定義為「動作 (actions)」。可以在設定對話框中為動作分配鍵盤快捷鍵。我們為許多動作提供了預設快捷鍵，但並非全部。

此測試版引入了新的動作列表。這提供了一種瀏覽和觸發任何動作的方法。

您可以從 `檢視 -> 動作列表` 功能表項開啟動作列表，或使用快捷鍵 `Ctrl-Shift-P`。

說到鍵盤快捷鍵，我們添加了一些新的預設值。

 - `Cmd/Ctrl-Y`：重做最後一次使用 `Cmd/Ctrl-z` 撤銷的編輯
 - `Cmd/Ctrl-Alt-L`：清除 Debug 側邊欄中的訊息
 - `Cmd/Ctrl-d`：部署您的流程

#### 視覺化 JSON 編輯器

JSON 編輯器現在提供了一種視覺化模式，您可以在其中編輯 JSON，而無需擔心引號是否放在正確的位置。

#### 新的接線技巧

在 0.20 中，在工作區中按住 Ctrl 並點擊會開啟快速新增對話框。現在，如果您在連線上按住 Ctrl 並點擊，則會顯示快速新增對話框，並且您新增的任何內容都將插入到您點擊的連線中。

#### 更多搜尋結果

搜尋對話框 (`Ctrl-f`) 以前僅返回前 25 個結果。這對某些擁有數百個標籤的使用者來說是個問題。該對話框現在可以顯示所有結果 — 雖然每次仍顯示 25 個。

#### 新的匯入/匯出對話框

剪貼簿對話框已與媒體庫匯入/匯出對話框合併，以提供更一致的流程匯入和匯出體驗。

各種類型的媒體庫（例如在 Function 節點中）也已更新為相匹配的樣式。

#### Context 側邊欄自動重新整理

Context 側邊欄現在有選項可以開啟或關閉隨編輯器中選擇變更而自動重新整理其內容的功能。

#### 選擇語言

使用者設定對話框下有一個新選項，可以選擇編輯器所呈現的語言。預設情況下，它將使用瀏覽器的偏好語言，但您可以覆蓋該設定以使用我們提供的其中一種語言：英語、日語、中文、韓語或德語。

如果您想幫助提供其他語言的翻譯，請與我們聯絡！

#### 編輯器 CSS 重組

我們全面改革了編輯器的內部樣式，為其提供了更一致的 CSS 結構。

*這可能是 Node-RED 歷史上最大的一次破壞性變更。這是那種我們只能在主版本號提升時才能做的變更。*

**這將破壞**：
 - 人們為 Node-RED 建立的任何自定義主題。
 - 超出我們提供的 API 範圍來自定義編輯器外觀的任何節點。

對此沒有簡單的解決辦法。在第一次嘗試此測試版之前，您可能需要停用自定義主題。

如果您正在使用精美的 `node-red-contrib-theme-midnight-red`，那麼已經有[支援 1.0 的版本可用](https://github.com/bonanitech/node-red-contrib-theme-midnight-red)。

編輯器使用的幾乎每一個 CSS 類別和 DOM ID 都已更改。我們以前使用的類別和 ID 反映了 5 年的增量開發，包括可以追溯到我開始建立 Node-RED 的第一天的內容。

到目前為止，它們運作得很好，但坦率地說，對於任何想要建立自定義主題或嘗試將編輯器嵌入到自己頁面中的人來說，它們是一團糟。

因此，此版本為所有這些內容帶來了更一致的命名方案。我們避免設定任何全域網頁樣式，而是僅將其應用於編輯器組件內的內容。

我們還重組了 CSS，使建立編輯器的自定義配色方案變得*容易得多*。我們還提供了一個指令碼，可以為您完成生成自定義主題的大部分工作。如果您有興趣，請到 Slack 與我們交流。

幾乎完整的變更清單可以在[這裡](https://docs.google.com/spreadsheets/d/1l-apmuM1mpUfvd2gUR__KD4SVkrkDJgsC8y0zyKk_bs/edit#gid=0)查看。

#### 編輯器依賴項

我們終於將 jQuery 和 jQuery-UI 的版本更新到了最新版 — 分別是 3.4.1 和 1.12.1。由於 jQuery 3.x 移除了我們以前依賴的一系列已棄用的 API，我們還包含了 jquery-migrate 以暫時保持運作。這還有一個額外的好處，即記錄了所有已棄用 API 的使用情況，以便我們可以開始找出需要更新的 contrib 節點。

在我們迎接最新版本 jQuery 的同時，我們也向 Bootstrap 告別。我們在編輯器的早期階段使用了 Bootstrap 主題，並且一直在慢慢擺脫它。在此版本中，最後殘留的 Bootstrap CSS 和 JavaScript 已被移除。


### 節點更新

核心節點進行了少量的錯誤修復和增強。

 - **CSV** 節點現在有一個選項可以關閉其對數值的智慧檢測，並將它們保留為字串。事實證明，它在某些邊緣情況下表現得過於智慧。
 - **Template** 節點*終於*有一個選項可以將編輯框展開為全螢幕編輯器，就像 Function 節點早已具備的功能一樣。
 - 現在更容易挑選 **Catch** 和 **Status** 節點所針對的節點。您現在可以切換回工作區並直接點擊您想要的節點，而無需僅從長長的節點列表中挑選。
 - **MQTT** 節點現在預設使用 3.1.1 版協議，而不是較舊的 3.1 版。我們目前還不支援 MQTTv5，但已有一個正在等待審核的 PR，敬請關注下一個版本。
 - **Switch** 節點有一個新的 "has key" 規則，用於檢查物件是否包含給定屬性，無論其值為何。
 - **HTTP Request** 節點現在允許您設定 keep-alive 選項，以便可以重複使用連接。


### 新的 Docker 映像檔

Docker 映像檔進行了大規模改革。我們現在為多種架構生成映像檔，並已遷移到 `node:XX-alpine` 基礎映像檔。

隨著這項變更，映像檔的名稱已更改為 `nodered/node-red`。我們不再向 `nodered/node-red-docker` 發佈更新。

查看 [Node-RED Docker 說明文件](https://github.com/node-red/node-red-docker/blob/master/README.md)以獲取更多詳細資訊。


### 流程庫 (Flow Library) 更新

伴隨著此版本的發佈，我們還對[流程庫 (Flow Library)](https://flows.nodered.org) 進行了重大更新。

除了新外觀外，我們還增加了對流程評分的能力，就像您以前可以對節點評分一樣。我們還引入了「收藏集 (Collections)」的概念；任何人都可以建立節點和流程的集合。

已經建立的一些收藏集包括：
 - [Node-RED Dashboard 的額外節點](https://flows.nodered.org/collection/590bc13ff3a5f005c7d2189bbb563976)
 - [幫助排程任務的節點和流程](https://flows.nodered.org/collection/c8156f6276976bfb518d1e60442e01e2)
 - [幫助路由訊息的節點](https://flows.nodered.org/collection/B1C6pe3gKjH9)

這是社群幫助管理媒體庫中所有精彩內容的好方法。
