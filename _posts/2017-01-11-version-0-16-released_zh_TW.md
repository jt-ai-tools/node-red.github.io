Node-RED 0.16 現在已經可以 [下載](https://github.com/node-red/node-red/releases/download/0.16.0/node-red-0.16.0.zip) 或透過 [npm 安裝](https://npmjs.org/package/node-red)。

如果升級，請閱讀 [升級說明](/docs/getting-started/upgrading_zh_TW.html)。

正如 0.15 版本所宣布的，0.16 將不會透過標準的 Raspberry Pi Jessie 儲存庫提供。請參閱下文了解如何將您的 Pi 更新到此版本的更多資訊。

---

### 不再支持 Node 0.10/0.12

正如在上一個版本中所宣布的，我們不再支持 Node.JS 0.10 或 0.12。

Node 專案已不再支持這兩個版本，繼續支持它們會阻礙我們獲取其他依賴項中的重要修復。

如果您在 Raspberry Pi 上運行預裝版本，則應按照 [此處](/docs/hardware/raspberrypi_zh_TW.html) 的說明升級您的 node.js 版本。

### node-red-node-serialport 已移除

正如我們在之前的版本中所宣布的，我們已將序列埠 (Serial) 節點從預設節點集中移除。

***如果您正在使用序列埠節點，升級後您將需要手動重新安裝該節點。***

### 編輯器更新

#### 快速添加節點/連線

Node-RED 有兩個基本操作 —— 添加節點並將其與另一個節點連線。

挑戰之一有時是在調色板中找到您想要的節點。這可能會導致大量的捲動 —— 尤其是如果您安裝了大量節點。

我們為節點增加了一種新的「快速添加」模式。如果您在工作區中按住 ctrl 點擊 (cmd-click)，將出現新的快速添加下拉選單。它在游標下方提供了一個迷你版的調色板，可以透過過濾來精確找到您正在尋找的節點。當您點擊該節點時，它就會出現在您的工作區中。

![](/blog/content/images/2017/01/quick-add.gif)

下拉選單頂部列出了五個常用的核心節點，然後是五個最近添加的節點，最後是所有按字母順序排列的節點。

我們已經使用這個功能一段時間了，發現它確實能加快建立流程的速度。您不必在操作過程中一直將焦點移回調色板。

另一個挑戰是當您在工作區中放置了一堆節點並想將它們全部連線起來時；需要大量的點擊拖動。

除了節點的快速添加模式外，我們也為連線做了類似的功能。

如果您在點擊節點端口開始連線時按住 ctrl (cmd)，然後您可以放開滑鼠按鈕，只需點擊您想要連線在一起的每個節點，最後放開 ctrl。

![](/blog/content/images/2017/01/quick-wire.gif)

由於兩種模式都是透過在點擊時按住 ctrl 鍵觸發的，因此您可以輕鬆地將它們結合使用。

![](/blog/content/images/2017/01/quick-add-and-wire.gif)

#### 更好的 Debug 面板

Debug 側邊欄進行了翻新，使其更加實用。以前它將訊息內容顯示為原始文字。對於簡單的文字或數值這沒問題，但一旦您發送對象 (Object) 或數組 (Array)，就很難在顯示的一大塊文字中看清細節。

新的側邊欄提供了更具結構化的訊息視圖。如果您熟悉瀏覽器中的 JavaScript 控制台，您就會熟悉此處的操作。

![](/blog/content/images/2017/01/debug.gif)

裡面還隱藏了一些額外的功能。例如，可以點擊任何數字類型來在十進位和十六進位之間切換。如果它看起來像是以自紀元以來的毫秒數表示的日期，再點擊一次將顯示格式化後的日期。

另一個例子是如果您收到一個實際上包含文字數據的 Buffer 對象：

![](/blog/content/images/2017/01/debug-string-buffer.gif)

您現在還可以透過點擊右下角的螢幕圖示將 debug 側邊欄彈出到其自己的窗口中。這讓您可以給偵錯訊息提供所需的充足空間。

![](/blog/content/images/2017/01/debug-detach.png)

如果您有任何其他讓 debug 面板更加實用的建議，請告訴我們。

#### 流程差異對比 (Flow diff)

我們在上一個版本中引入了一個警告，提醒您即將覆蓋運行時上的更改。我們知道該警告的作用有限 —— 它無法幫助您採取任何行動。

此版本現在讓您可以將伺服器上的流程與您在本地所做的更改進行比較。然後您可以將這些更改合併在一起，並在此過程中解決任何衝突。

![](/blog/content/images/2017/01/diff-view.png)


#### 新的鍵盤快捷鍵

編輯器中的鍵盤處理進行了翻新，以便更容易為任何操作添加鍵盤快捷鍵。您可以透過按 `Shift-?` 或從相應的選單選項中查看可用鍵盤快捷鍵的完整列表。

一些重點包括：

 - 打開搜尋對話框已移至 `Ctrl-F` —— 這消除了與 Windows 系統快捷鍵的衝突，而且總體上更加合理。
 - `Ctrl-G` 後跟 `i`、`d` 或 `c` 將分別打開 info、debug 或配置節點側邊欄標籤頁。
 - 您可以使用 `Ctrl-Shift-j` 和 `Ctrl-Shift-k` 切換標籤頁。

該列表還顯示了目前尚未分配快捷鍵的其他已定義操作；這項工作是允許您在未來版本中自定義快捷鍵的前奏。

#### 運行時狀態通知

我們在編輯器中增加了一個警告，會在您的流程因缺少節點類型而停止時通知您。這曾是不看運行時日誌的使用者感到困惑的原因。

![](/blog/content/images/2017/01/stopped-warning.png)

#### JSONata

[JSONata](http://jsonata.org/) 是一種輕量級的查詢和轉換語言，用於處理 JSON 文件，由我們在 IBM 的一些同事開發。

我們在 Change 和 Switch 節點中增加了對它的支持。這意味著許多簡單的任務現在可以在不訴諸 Function 節點的情況下完成。

例如，一個將 `msg.payload` 從華氏度轉換為攝氏度的 Change 節點：

![](/blog/content/images/2017/01/jsonata-change.png)

為了幫助您編寫 JSONata 表達式，您可以透過點擊欄位右側的按鈕打開表達式編輯器。這為完整的 JSONata 語言提供了自動完成和行內幫助。

有關 JSONata 的更多資訊，請訪問他們的 [網站](http://jsonata.org/) 並使用他們的 [練習器來試用它](http://try.jsonata.org/)。


#### 其他變更

 - 我們在 info 側邊欄中增加了一組使用編輯器的提示 —— 並非每個人都會閱讀版本說明。
 - 使用調色板管理員安裝或移除節點現在有一個額外的步驟來確認操作。
 - 許多節點使用的 `typedInput` 組件現在支援鍵盤操作。
 - 我們在節點編輯對話框中增加了一個 `delete` 按鈕 —— 並非每個人都意識到可以使用 `delete` 鍵執行相同的操作。


### 節點更新

 - 增加了一個新的 `YAML` 格式解析器節點。
 - `template` 節點現在可以引用 `global` 和 `flow` 上下文屬性 —— 查看其更新後的 info 標籤頁以獲取範例。
 - Switch 節點現在在您重新排列規則時會保持您的連線順序。
