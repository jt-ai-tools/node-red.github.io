Node-RED 0.14.3 現在已經可以 [下載](https://github.com/node-red/node-red/releases/download/0.14.3/node-red-0.14.3.zip) 或透過 [npm 安裝](https://npmjs.org/package/node-red)。

如果升級，請閱讀 [升級說明](/docs/getting-started/upgrading_zh_TW.html)。

如果您使用的是 Raspberry Pi Jessie 上的預裝版本，儲存庫將在不久的將來更新。

---

### 編輯器更新

此版本編輯器最大的變化是新的節點編輯托盤 (edit tray)，它取代了舊的編輯對話框。這是我們工作了一段時間的事情，也是啟用此版本中某些其他功能所必需的變化。

由於這是一個巨大的變化，我們在幾週前向社群提供了一個演示。
這帶來了許多很棒的反饋，幫助我們在過程中改進了設計。

![](/blog/content/images/2016/06/edit-tray.gif)

舊編輯對話框的一個非常常見的投訴是在編輯器打開時無法與側邊欄交互；而這正是您最想查看節點幫助的時候。有了新的編輯托盤，您可以在編輯節點的同時繼續與 Info 和 Debug 標籤頁交互。

#### 分離連線 (再次提報)

上一個版本增加了在拖動節點端口時按住 Shift 鍵來移動連線的功能。我們在此版本中對其進行了更新，如果當前選中了其中一條連線，我們將只移動該連線，而不是整個束。

![](/blog/content/images/2016/06/move-link.gif)

#### 重新排列標籤頁

是的 —— 重新排列您的標籤頁。不客氣。

![](/blog/content/images/2016/06/tabs-order.gif)

#### 過濾 Debug 側邊欄

您現在可以過濾 debug 側邊欄，僅顯示來自當前標籤頁上節點的偵錯消息。啟用後，當您切換標籤頁時，側邊欄會刷新顯示您所在標籤頁的消息。

此外，如果消息來自子流程內部的 debug 節點，我們將突出顯示子流程實例節點以幫助您追蹤它。

#### 內建變更日誌 (Change-log)

有這麼多變更，使用者很難知道發生了什麼變化。信不信由你，並非每個人都會閱讀這些部落格文章。我們現在在下拉選單中提供了列出版本中所有高級變更的變更日誌。

這是幫助使用者發現新功能的一小步。

### 節點更新

#### Link 節點

我們增加了一對新節點，可以幫助您更好地組織流程；Link 節點讓您可以建立一條從一個流程末尾到另一個流程開頭的虛擬連線。

如果您選中一個 link 節點，它將在您的工作區顯示虛擬連線，包括任何消失到另一個標籤頁的鏈接。為了幫助您在它們之間導航，可以點擊指向另一個標籤頁的鏈接，您將被帶到該標籤頁，並且相應的 link 節點會被預先選中。

![](/blog/content/images/2016/06/link-nodes.gif)

#### Split/Join 節點

這些功能已經在待辦清單上很久了，我們終於在一個版本中實現了它們。

Split 節點可用於將單個消息拆分為一系列消息。例如，如果 `msg.payload` 是字串，它將使用您想要的任何字符拆分該文本，從而為每個部分獲取一條消息。數組將被拆分為其各個元素，對象將被拆分為其鍵/值對。

Join 節點預設情況下會執行與 Split 節點相反的操作。如果您向它提供由 Split 節點發送的一系列消息，它將把它們重新組合回其原始形式。您還可以為各種場景手動配置它。查看節點幫助以獲取更多信息。

#### 支持 TLS 的 MQTT

MQTT 節點現在可以建立配置正確的 TLS 連接。這使用了一個新的通用 TLS 配置節點，您可以在其中提供證書文件的路徑。此節點可供其他節點開始使用 —— 查看 mqtt-broker 節點以了解如何使用它。

在處理這些內容時，我們增加了在 MQTT In 節點上設置您想要訂閱的 QoS 的功能。

#### HTTP In 節點 —— Cookie 支持和 HTTP PATCH

HTTP In 節點進行了兩項重要更新：增加了 PATCH 選項，並在堆棧中增加了標準的 cookie 解析器中間件。

這第二項意味著您現在可以使用 `msg.req.cookies` 存取會話上設置的任何 cookie。

我們還使通過傳遞給 HTTP Response 節點的消息上的 `msg.cookies` 屬性來設置/清除 cookie 變得更加容易。查看節點信息了解更多詳情。

#### Change 節點

Change 節點增加了一個新選項來移動屬性 —— 以前您可以通過「設置」後跟「刪除」兩步來完成。

我們還更新了識別屬性時可以使用的語法，以使用更完整的對象語法。例如，您現在可以存取屬性，如 `msg.payload[1].name` 或 `msg.payload["another property"]`。這對一些節點使用的任何 TypedInput 都是適用的。
