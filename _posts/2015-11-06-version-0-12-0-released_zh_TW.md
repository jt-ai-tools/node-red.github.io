### 更新至 Express 4.x

Express 是我們用來提供編輯器、管理 API 和 HTTP 節點的 Web 框架。我們自開始以來就一直使用 Express 3.x，但該版本已不再維護，因此我們必須升級到最新的穩定版本 4.x。像這樣的重大版本號提升標示著庫中一些不兼容的 API 變更，總體而言，我們可以對 Node-RED 使用者隱藏這些變更。但如果您將 Node-RED 嵌入到您自己的應用程式中，並提供您自己的 Express 實例，如果版本不匹配，可能會出現問題。

我以前在郵件列表中說過，我對 Node-RED 早期選擇的一個遺憾是我們暴露了 `HTTP In` 節點發送的訊息上的原始 Express 請求 (request) 和響應 (response) 對象。我們希望達到可以可靠地將訊息對象編碼和解碼為 JSON 格式的程度 —— 這在今天附帶這些對象的情況下是做不到的。因此，在這次發佈中，我們棄用了使用這些對象提供的任何函數。它們在此版本中仍可工作，但您將在日誌中看到警告訊息。大多數函數都有可以使用的替代方案 —— 如果您不確定，請到 [郵件列表](https://groups.google.com/forum/#!forum/node-red) 詢問。

### 支持 Node 4.x

隨著 node.js 和 io.js 的重新統一，[Node v4.2.0](https://nodejs.org/en/blog/release/v4.2.0/) 發佈了，這是一個長期支持 (LTS) 版本。

核心 Node-RED 運行時現在支持在 4.x 版本的 node 上運行 —— 但個別節點可能需要一些時間才能趕上，因為 4.x 對二進位模組的構建方式帶來了顯著變化。

例如，我們的 Serial 節點依賴於 `serialport` 模組。他們已經發佈了一個支持 4.x 的新版本，但它需要比 Raspberry Pi 預設提供的更高版本的 GCC 編譯器。因此安裝它需要的不僅僅是 `npm update`。一旦我們弄清楚了最好的方法，我們就會將其記錄並分享。


### 配置節點 (Configuration Nodes)

配置節點（提醒一下）是物理上不出現在流程中，但包含可以由流程中的節點重複使用的共享配置的節點。例如，`mqtt-broker` 配置節點包含可以由多個 `mqtt in` 和 `mqtt out` 節點使用的代理 (broker) 的所有連接詳細信息。

到目前為止，配置節點浮動在全域層級，可以由任何標籤頁或子流程中的任何節點使用。在此版本中，配置節點現在可以限定為特定的標籤頁或子流程，或者保留為全域節點。

這意味著如果您導入一個包含一些配置節點的子流程，然後從調色板中刪除該子流程，配置節點也將被刪除 —— 它們不會像以前那樣留在全域層級。

配置節點的編輯對話框允許您在右上角選擇節點的範圍 —— 並且會警告您更改範圍是否會使其對目前使用它的節點不可用。

### Status 節點

節點已經有一段時間能夠分享狀態消息了 —— 例如 MQTT 節點指示它們是否已連接。所缺少的是一種流程對這些消息做出反應的方式。

新的 Status 節點允許您做到這一點。概念上與上一個版本中的 Catch 節點類似，當同一標籤頁上的節點更新其狀態時，Status 節點將被觸發。

因為一個標籤頁可能有多個節點具有狀態更新，Status 節點可以針對標籤頁上的特定節點 —— 使得為特定場景建立特定流程變得更加容易。因為這個針對性功能非常有用，我們也更新了 Catch 節點，使其能夠執行相同的操作。

### MQTT 節點

MQTT 節點進行了遲來的更新，以支持協議的更多功能，包括：

 - Keep-alive 計時器和清除會話 (clean session) 設置
 - TLS 連接*
 - Birth 和 Will 消息（在連接或連接丟失時自動發布消息）。

目前的 TLS 支持非常基礎 —— 您不能指定自己的證書，因此您目前可能需要禁用驗證服務器證書。但我們知道我們需要為任何基於 TLS 的連接提供更好的全方位證書管理方式。在達到目標之前還有一些障礙需要克服，但請期待進一步的改進。


### 節點更新

一如既往，許多其他節點進行了各種更新。

- JSON 節點可以編碼 Array 類型
- Switch 節點正則表達式規則現在可以設置為不區分大小寫
- HTTP In 節點可以接受非 UTF8 payload —— 在適當情況下將返回 Buffer
- 不論 spawn 選項如何，Exec 節點配置保持一致
- Function 節點現在可以顯示狀態圖示/文本
- CSV 節點現在可以處理數組
- setInterval/clearInterval 已添加到 Function 節點中
- 當節點停止時，Function 節點會自動清除所有計時器 (setInterval/setTimeout)
